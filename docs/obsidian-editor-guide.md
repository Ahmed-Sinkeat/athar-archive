# دليل المحرِّر: تحرير موقع «أهل الأثر» عبر أوبسيديان (Obsidian)

هذا الدليل لمن يريد إضافة الكتب وتراجم الأعلام وتعديلها. لا تحتاج أي خبرة برمجية —
كل مادة في الموقع هي ملف نصي بسيط، وأوبسيديان محرِّر ملفات، وGitHub هو المكان الذي
تُحفَظ فيه الملفات ويشاركها الجميع.

> **الفكرة باختصار:** تفتح أوبسيديان → تكتب أو تعدِّل → تضغط زر المزامنة →
> يصل التعديل إلى GitHub → يراجعه أحمد وينشره على الموقع.

---

## أولًا: التجهيز (مرة واحدة فقط)

### ١. حساب GitHub وقبول الدعوة
1. أنشئ حسابًا مجانيًا في [github.com](https://github.com) إن لم يكن عندك.
2. أخبر أحمد باسم حسابك ليرسل لك دعوة تعاون على المستودع.
3. **مهم:** افتح بريدك الإلكتروني واضغط قبول الدعوة (Accept invitation) —
   بدون هذه الخطوة لن يعمل شيء.

### ٢. تثبيت برنامج Git
1. حمِّله من [git-scm.com](https://git-scm.com) وثبِّته.
2. اقبل الخيارات الافتراضية كلها أثناء التثبيت (التالي، التالي… إنهاء).

### ٣. تثبيت أوبسيديان وإضافة Git
1. حمِّل أوبسيديان من [obsidian.md](https://obsidian.md) وثبِّته.
2. افتح الإعدادات (رمز الترس) → **Community plugins** → عطِّل الوضع المقيَّد
   (Turn off Restricted mode).
3. اضغط **Browse** وابحث عن إضافة اسمها **Git** (للمطوِّر Vinzent) →
   **Install** ثم **Enable**.

### ٤. جلب نسخة الموقع
1. اضغط `Ctrl+P` (لوحة الأوامر) واكتب `Clone`.
2. اختر **Git: Clone an existing remote repo**.
3. الصق هذا الرابط:
   `https://github.com/Ahmed-Sinkeat/athar-archive.git`
4. عند أول اتصال ستُفتح نافذة متصفح لتسجيل الدخول إلى GitHub — سجِّل بحسابك
   الذي قَبِل الدعوة، مرة واحدة ويُحفَظ.
5. انتظر اكتمال التنزيل (أول مرة تأخذ دقائق لأن الموقع كبير).

### ٥. إعدادات الإضافة المريحة
في الإعدادات → **Git**:
- **Pull on startup**: فعِّلها (يجلب آخر نسخة تلقائيًا عند فتح البرنامج).
- **Auto commit-and-sync**: فعِّلها واجعلها كل ١٠ دقائق — وبهذا كل ما تكتبه
  يُرفَع وحده دون أي خطوة إضافية. (أو اتركها معطَّلة وارفع يدويًا كما في الفقرة
  التالية).

## ثانيًا: العمل اليومي

ملفاتك كلها داخل مجلد `src/content/`:

| المجلد | ماذا فيه |
|---|---|
| `src/content/book/` | الكتب |
| `src/content/person/` | تراجم الأعلام |

**لا تعدِّل خارج هذين المجلدين.**

### تعديل مادة موجودة
1. افتح أوبسيديان (سيجلب آخر نسخة وحده إذا فعَّلت Pull on startup).
2. ابحث عن الملف: `Ctrl+O` ثم اكتب اسم الكتاب أو العَلَم.
3. عدِّل واحفظ (`Ctrl+S`).
4. إن لم تكن المزامنة التلقائية مفعَّلة: افتح لوحة **Source Control** من الشريط
   الجانبي (رمز التفرُّع) → اكتب وصفًا قصيرًا لتعديلك → اضغط
   **Commit-and-sync**.

### إضافة كتاب جديد
1. أنشئ ملفًا جديدًا داخل `src/content/book/`.
2. اسم الملف بالإنجليزية، حروف صغيرة وأرقام وشرطات فقط — هذا الاسم يصبح رابط
   الصفحة ولا يتغيَّر بعد النشر. مثال: `lumat-al-itiqad.md`.
3. ابدأ الملف بهذه الترويسة (بين علامتي `---`) ثم اكتب نص الكتاب بعدها:

```markdown
---
title: "لمعة الاعتقاد"
status: draft
published_at: 2026-07-15
person: ibn-qudamah
topics: ["al-aqeedah-al-aamah"]
description: "وصف موجز للكتاب في جملتين تظهران في نتائج البحث."
---

نص الكتاب هنا…
```

شرح الحقول:

| الحقل | معناه |
|---|---|
| `title` | عنوان الكتاب بالعربية |
| `status` | `draft` = مسودة لا تظهر في الموقع، `published` = منشور. ابدأ دائمًا بـ `draft` |
| `published_at` | تاريخ اليوم بصيغة سنة-شهر-يوم |
| `person` | المؤلف — **اسم ملفه** في مجلد `person/` (انظر الفقرة التالية) |
| `topics` | موضوع الكتاب — اختياري |
| `description` | تعريف موجز بالكتاب — اختياري لكنه مفيد جدًّا لمحركات البحث |

### كيف أعرف قيمة `person`؟
قيمة المؤلف هي اسم ملفه داخل مجلد `src/content/person/` (بلا `.md`).
طريقتان لمعرفتها:
- تصفَّح مجلد `person/` في أوبسيديان وانظر اسم الملف، مثل `ibn-qudamah.md`
  → القيمة `ibn-qudamah`.
- أو افتح صفحة العَلَم في الموقع وانظر الرابط:
  `athar.arthurarchive.com/person/ibn-qudamah` → القيمة `ibn-qudamah`.

إن لم يكن للمؤلف ترجمة بعد، أنشئ له ملفًا في `person/` أولًا:

```markdown
---
title: "الموفق ابن قدامة المقدسي"
status: published
published_at: 2026-07-15
died: "620 هـ"
bio: "سطر أو سطران في التعريف به."
---
```

## ثالثًا: قواعد مهمة

1. **افتح البرنامج واترك المزامنة تجلب الجديد قبل أن تكتب** — هذا يمنع تعارض
   النسخ. وإذا ظهر ملف فيه علامات غريبة مثل `<<<<<<<` فلا تحذفها بنفسك؛ أخبر
   أحمد.
2. **لا يعمل اثنان على الملف نفسه في الوقت نفسه.**
3. **الملفات الصوتية الكبيرة لا تُرفَع هنا أبدًا** — أرسلها إلى أحمد وهو يرفعها
   إلى المخزن ويعطيك الرابط.
4. ما ترفعه لا يظهر في الموقع فورًا: أحمد يراجع ثم ينشر. والموقع نفسه يفحص كل
   ملف قبل النشر، فلو أخطأت في حقلٍ ما فلن يتعطَّل شيء — سيُكتشَف الخطأ قبل
   النشر ويُصحَّح.
5. اجعل الحالة `draft` حتى يكتمل الكتاب، ثم غيِّرها إلى `published`.
6. استخدم أوبسيديان على الحاسوب فقط، لا على الجوال (المستودع كبير على تطبيق
   الجوال).

## أسئلة سريعة

**نسيت هل رُفع تعديلي؟** افتح لوحة Source Control — إن كانت فارغة فكل شيء
مرفوع.

**أخطأت وخرَّبت ملفًا؟** لا تقلق، كل نسخة قديمة محفوظة في GitHub ويمكن
استرجاعها. أخبر أحمد فقط.

**أريد كتابة عناوين وفقرات داخل الكتاب؟** استخدم صيغة ماركداون البسيطة:
`## عنوان الباب` لعنوان، وسطر فارغ بين الفقرات، و`**نص**` للتغميق.
