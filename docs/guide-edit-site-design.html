Editing the Site's Look

Everything on the site that isn't book/poem/lesson content — logo, favicon, colors, fonts, menu labels, footer — lives in a small set of code files. This guide points to each one.

Golden rule: only change the readable Arabic/English text or the file being replaced. Never touch anything between < > or { }, or CSS variable names.

1. Logo & favicon

WhatFileHow to change
Hero logo (light mode)public/calligraphy/logo_light.webpReplace this file directly, keep the same filename and roughly the same square shape (shown at 560×560).
Hero logo (dark mode)public/calligraphy/logo_dark.webpSame as above — this one shows automatically when the visitor's system is in dark mode.
Site name text (header/footer, not the hero image)src/layouts/Base.astro (~line 71, 84, ~226)Find "أهل الأثر" and replace the text in all 3 places.
Favicon (browser tab icon)public/favicon.svgReplace this file directly with your new icon (keep the filename favicon.svg).

The hero logo markup itself is in src/pages/index.astro (~line 107) if you ever need to resize it or change the alt text.

2. Colors, fonts, themes

All in one file: src/styles/global.css, at the top (:root block, line ~9).

--brand: #7A2222;   /* main maroon */
--brand2: ...;      /* secondary brand color */
--gold: #8E6A1C;    /* accent gold */
--f-disp: 'Amiri';               /* headings */
--f-quran: 'Amiri Quran';        /* Quran text */
--f-ui: 'IBM Plex Sans Arabic';  /* body / UI text */

The site has 3 themes (paper / noir / mono) — each one redefines these same variables further down the file (~line 58, ~68). Change a color in all three places if you want it consistent across themes.

Home page, browse pages, and reader chrome text live in their own files under src/pages/ — see docs/editing-text.en.md for the full per-page table.

How to make a change

1. Open the repo on GitHub, find the file above.
2. Click the pencil ✏️ (Edit).
3. Change only the value/text you intend to (a color hex, a font name, a label).
4. Scroll down, click Commit changes — the site rebuilds automatically in a few minutes.

Mistake? GitHub keeps full history — revert the commit, no harm done.
