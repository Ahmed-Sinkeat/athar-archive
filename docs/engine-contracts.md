# Athar Engine — API & Schema Contracts

> **Status:** Accepted 2026-06-27. This document specifies the unified Public API contracts and shared schema definitions for Athar Engine. Any client application (like the Ahl al-Athar website or Android App) interacts with the engine exclusively through these contracts.

---

## 1. The Public API Contract
The public interface of `Athar Engine` is composed of 5 pipeline orchestration functions. Clients call these functions in sequence or package them together within an intake pipeline.

```typescript
interface AtharEngine {
  /**
   * 1. Ingests raw binary files and translates them to a unified document structure.
   */
  parseDocument(file: RawFile, options?: IngressOptions): Promise<Document>;

  /**
   * 2. Runs the Rules Engine on a Document to extract headings and semantic tokens.
   */
  extractKnowledge(doc: Document, rules: RuleProfile): Promise<SemanticAST>;

  /**
   * 3. Matches extracted tokens against the Entity Registry to resolve identities and resolve ambiguity.
   */
  enrichEntities(ast: SemanticAST, registry: EntityRegistry): Promise<EnrichedAST>;

  /**
   * 4. Serializes an enriched AST to the canonical diacritized Markdown layout.
   */
  renderMarkdown(ast: EnrichedAST): Promise<string>;

  /**
   * 5. Runs cross-entity integrity tests over a folder of compiled resources.
   */
  validateCorpus(corpusPath: string): Promise<ValidationError[]>;
}
```

## 2. Structural Contracts (Schemas)
These schemas define the shared data objects (contracts) that pass between modules of the engine.

### 2.1. `Document`
The raw representation of ingested files, stripped of binary layouts but preserving structural chunks:
```typescript
interface Document {
  metadata: {
    title: string;
    author?: string;
    sourceFormat: "epub" | "docx" | "html" | "pdf";
    ingestedAt: string;
  };
  chapters: Array<{
    title: string;
    level: number;
    paragraphs: string[];
    footnotes: Record<string, string>; // Maps index (e.g., "[1]") to footnote text
  }>;
}
```

### 2.2. `SemanticNode`
The ephemeral Abstract Syntax Tree nodes generated by extractors:
```typescript
type NodeType = 
  | "Book" | "Chapter" | "Section" | "Heading" | "Paragraph" 
  | "QuranVerse" | "Hadith" | "ScholarMention" | "BookReference" | "TermMention";

interface SemanticNode {
  type: NodeType;
  content?: string;                     // Raw matched text span
  attributes?: Record<string, any>;     // Extracted attributes (e.g. surah, ayah, resolved ID)
  children: SemanticNode[];
  confidence?: number;                  // 0.0 to 1.0 match trust score
  origin?: {
    page?: number;
    offsetStart?: number;
    offsetEnd?: number;
  };
}
```

### 2.3. `KnowledgeEntity`
The records stored in the canonical `Entity Registry`:
```typescript
interface KnowledgeEntity {
  id: string;                           // Canonical ID (e.g., "scholar:00001842", "place:00000091")
  slug: string;                         // Latin URL alias
  type: "scholar" | "book" | "place" | "sect" | "topic" | "domain";
  names: {
    formal: string;                     // Full official Arabic name
    common: string;                     // Popular Arabic common name
    english?: string;                   // Simplified English transliteration
  };
  aliases: string[];                    // Synonyms/Kunyate for lookup
  attributes: Record<string, any>;     // Type-specific attributes (e.g., died, madhhab, coordinates)
}
```

### 2.4. `KnowledgeRelation`
Graph relationships extracted by the engine:
```typescript
interface KnowledgeRelation {
  sourceId: string;                     // Source entity or resource ID
  targetId: string;                     // Target entity or resource ID
  relationType: "mentions" | "quotes" | "references" | "cites" | "narrates_from" | "student_of" | "lived_in";
  contextExcerpt?: string;              // Context sentence snippet where the link occurs
  confidence: number;                   // Resolution accuracy trust score
}
```

## 3. Extensibility Interfaces

Any importer or pattern extractor module written for the engine must satisfy these standardized interfaces.

### 3.1. `Importer`
Handles file reading and conversion to a raw `Document`:
```typescript
interface Importer {
  supportedExtensions: string[];
  import(fileBytes: Uint8Array): Promise<Document>;
}
```

### 3.2. `Extractor`
Processes text nodes and inserts parsed `SemanticNode` splits:
```typescript
interface Extractor {
  ruleProfile: string;
  extract(node: SemanticNode): Promise<SemanticNode[]>;
}
```
